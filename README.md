# 카드사 명세서 → 가계부 자동 정리

카드사별 .xls 명세서 파일을 업로드하면 거래일자/가맹점명/금액/카테고리 4가지를 추출해서 이미지 양식(MJ좌측·HJ우측 나란히)으로 정리하는 워크플로우.

## 사용 방법

### 방법 A. 웹앱 (권장 — 매월 셀프서비스)

1. `가계부_변환기.html` 을 더블클릭해서 브라우저로 열기 (인터넷 연결 필요 — CDN에서 SheetJS 로드)
2. **MJ 카드 / HJ 카드** 라디오 버튼으로 사용자 선택
3. 카드 명세서(.xls / .xlsx) 파일을 드래그 또는 클릭으로 업로드 → 여러 파일 한꺼번에 가능, 카드사 자동 감지
4. (선택) 국민카드의 쿠팡/네이버 결제 상품명을 보강하려면 "쿠팡 주문 데이터" 영역에 구글시트 데이터를 CSV로 붙여넣고 매칭
5. 거래 내역 테이블에서 가맹점명/카테고리 셀을 클릭해서 직접 수정 가능 (× 버튼으로 행 삭제)
6. `xlsx 다운로드` 버튼 → 이미지 양식의 xlsx 파일이 자동 저장됨. `표 복사` 버튼은 구글시트에 바로 붙여넣을 TSV 클립보드 복사.

### 방법 B. Python 스크립트 (자동화/배치용)

- `../outputs/budget/parse_cards.py` — 카드사별 .xls 파싱 + 쿠팡 매칭 + 카테고리 분류
- `../outputs/budget/build_xlsx.py` — 결과 xlsx 생성기

## 폴더 구조

- `가계부_변환기.html` — **메인 웹앱** (단일 파일, 오프라인 보관 가능)
- `2026-05_가계부.xlsx` — 5월 결과 파일
- `2026-05_가계부.html` — 정적 보기용 페이지

## 지원 카드사

| 카드사 | 파일 형식 | 가맹점명 |
|---|---|---|
| 현대카드 | HTML(.xls 확장자) | 실제 가맹점 ✓ |
| 국민카드 | OLE2(.xls) | 쿠팡/네이버페이는 PG명만 → 쿠팡 시트로 매칭 보강 |
| 하나카드 | OLE2(.xls) | 실제 가맹점 ✓ |

## 매월 처리 순서

1. 카드사 홈페이지에서 명세서를 .xls로 다운로드
2. 파일명을 `(카드사2글자)(월)월.xls` 로: `현카5월.xls`, `국카5월.xls`, `하나5월.xls`
3. 업로드한 뒤 "5월 가계부 정리해줘"
4. `parse_cards.py` 가 명세서를 파싱하고, 국민카드의 쿠팡/네이버 결제는 쿠팡 주문 시트와 (날짜+합계금액) 매칭으로 상품명을 채워줌
5. `build_xlsx.py` 가 결과 xlsx 3개 시트 생성

결과 xlsx 시트:
- `2026-05` — 메인. MJ 좌측(A~D), HJ 우측(F~I), 1행에 합계 SUM
- `카테고리요약` — 카테고리별 금액·건수
- `카드별합계` — 카드사별 금액·건수

## 알려진 제약/한계

1. **국민카드 가맹점 매칭 미스 9건**: 쿠팡 주문 시트에 없거나(쿠팡이츠 배달 등) 부분배송·배송비 합산 등으로 금액이 안 맞는 케이스. 명세서엔 그대로 `쿠팡(쿠페이)` / `네이버페이`로 남음. 필요하면 가맹점명 셀을 직접 수정.
2. **이미지 양식(첨부)의 MJ 합계 2,280,043원과 차이**: 이번 추출은 2,148,233원으로 약 13만원 차이. 다른 카드(예: 신한 등) 명세서가 더 있거나, 이미지가 다른 기간 데이터일 가능성. 확인 후 명세서 추가하면 자동 합산됨.
3. **HJ 데이터**: 이번 5월은 MJ 카드 3장만. HJ 명세서 업로드 시 우측(F~I) 영역으로 채울 예정.
4. **국민카드 '전표미매입' 상태 거래**: 카드사 매입 처리 전이라도 실제 결제는 발생했으므로 포함했음. 카드사 청구 시점과 다를 수 있음.
