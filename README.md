# 가계부 변환기 (뱅크샐러드 xlsx)

뱅크샐러드에서 받은 xlsx 파일 ("가계부 내역" 시트 포함)을 업로드해서 월별 시트로 정리하는 웹앱.

## 사용 방법

1. [`가계부_변환기.html`](가계부_변환기.html)을 더블클릭해서 브라우저로 열기 (인터넷 연결 필요 — CDN에서 SheetJS 로드).
2. **MJ / HJ** 라디오 버튼으로 사용자 선택. 두 사람 파일을 따로 업로드하면 좌(MJ)/우(HJ)로 함께 표시됨.
3. xlsx 파일 드래그 또는 클릭해서 업로드.
4. 처리 옵션 체크 (기본값: 지출만, 자동 상쇄 ON).
5. 월별 탭에서 해당 월 거래 확인. 가맹점명·카테고리 셀 클릭해서 직접 수정 가능, × 버튼으로 행 삭제.
6. `xlsx 다운로드` 버튼 → `가계부_<시작월>_<종료월>.xlsx` 자동 저장. `현재 월 표 복사` 버튼은 TSV 클립보드 복사 (구글시트 붙여넣기용).

## 파일 양식

업로드할 xlsx에는 "가계부 내역" 같은 시트가 있어야 하고, 그 시트 첫 행 헤더는 다음 컬럼을 포함해야 함 (순서 무관):

| 컬럼명 | 의미 |
|---|---|
| 날짜 | YYYY-MM-DD 또는 Excel 날짜 |
| 시간 | HH:MM:SS (선택) |
| 타입 | 지출 / 수입 / 이체 |
| 대분류 | 식비, 자동차, 온라인쇼핑 등 |
| 소분류 | 배달, 주유, 인터넷쇼핑 등 |
| 내용 | 가맹점명 또는 거래 설명 |
| 금액 | 음수=지출, 양수=수입/환불 |
| 화폐 | KRW 등 (선택) |
| 결제수단 | 카드명 / 결제수단 |
| 메모 | 자유 텍스트 (선택) |

뱅크샐러드 기본 다운로드 양식 그대로 호환됨.

## 처리 옵션

### 양수/음수 쌍 자동 상쇄 (기본 ON)

같은 (날짜·내용·절대값·대분류·소분류·결제수단) 키의 양수 1개 + 음수 1개를 쌍으로 인식해서 두 행 모두 제거. 카드 결제 → 취소 → 재결제 같은 패턴의 환불·취소 거래를 깔끔하게 정리.

예시:
- 2026-05-20 10:17 새서울석유 −150,000원 (1차 결제)
- 2026-05-20 10:19 새서울석유 +150,000원 (취소)
- 2026-05-20 10:19 새서울석유 −109,389원 (재결제)

상쇄 후 −109,389원만 남음.

### 수입 / 이체 포함 (기본 OFF)

- '수입' = 급여·환불·이자 등. 가계부에서 메인 표에 포함할지 선택.
- '이체' = 내계좌이체·카드대금·저축 등 자금 이동. 자산 위치만 바뀌는 거라 보통 제외.

### 자동 중복 제거 (업로드 시 항상 작동)

기존에 업로드한 거래와 중복되는 새 거래는 자동으로 걸러져요. 같은 xlsx를 두 번 올리거나, 기간이 겹치는 새 명세서를 올려도 중복 누적되지 않음.

- **판정 키**: `사용자(MJ/HJ) | 날짜 | 시간 | 내용 | 금액 | 대분류 | 소분류 | 결제수단`
- **메모(memo)는 키에서 제외** — 뱅크샐러드에서 메모를 수정해도 같은 거래로 인식.
- **기존 데이터 보존**: 키 충돌 시 새 레코드를 버려서, 사용자가 화면에서 직접 수정한 내용이 그대로 유지됨.
- 토스트 메시지로 추가/제외 건수 표시 (예: `✅ 파일명 → MJ: 12건 추가, 8건 중복 제외`).

## 결과 xlsx 구조

월별로 시트 하나씩 (`YYYY-MM` 이름) — 각 월별 이미지 양식.
- A~D: MJ (거래일자 / 가맹점명 / 금액 / 대분류·소분류)
- F~I: HJ (동일 구조)
- 1행 머리에 이름과 월 합계

## 구글시트 동기화 (선택)

웹앱의 "5. 구글시트 동기화" 섹션을 통해 Apps Script 웹앱으로 데이터를 직접 보낼 수 있어요.

### 일회성 설정 (5분)

1. 새 구글시트를 만든다.
2. 메뉴 **확장 프로그램 → Apps Script** 클릭.
3. 기본 코드를 모두 지우고 같은 폴더의 [`apps_script.gs`](apps_script.gs) 내용을 통째로 붙여넣고 저장(Ctrl+S).
4. 오른쪽 상단 **배포 → 새 배포** 클릭 → 유형은 **웹 앱**.
   - 실행: `본인 (내 계정)`
   - 액세스: `본인만` (또는 본인이 원하는 범위)
5. 처음 한 번은 권한 승인 절차: "고급 → 안전하지 않은 페이지로 이동" → 계정 선택 → 허용.
6. 배포 완료 후 나오는 **웹 앱 URL** 복사 → 가계부_변환기.html의 입력란에 붙여넣고 `URL 저장`.

### 매월 사용

1. 새 명세서 xlsx 업로드 → 화면 결과 확인.
2. 두 가지 동기화 버튼 중 선택:
   - `🔄 전체 동기화`: 업로드한 모든 월의 시트를 한 번에 갱신
   - `📌 현재 월만 동기화`: 화면에서 활성화된 탭의 월 한 개만 갱신
3. 1~2초 후 구글시트 새로고침해서 결과 확인.

### 동기화 동작 (MJ/HJ 분리 + append-only)

- **시트 분리**: 사용자별로 별도 시트 생성. 시트명은 `MJ2026-05`, `HJ2026-05` 형식 (사용자 prefix + 월).
- **컬럼 5열 구조** (시트 한 장당):
  - A=거래일자
  - B=이름(가맹점명)
  - C=이름(중복 — 사용자가 자유롭게 수정 가능, 동기화 시 절대 안 건드림)
  - D=금액
  - E=카테고리
- **2026년만 동기화**: 2025년 데이터는 동기화 대상에서 제외. (화면에는 표시되지만 시트에는 안 들어감.)
- **append-only**: 기존 시트 행은 절대 덮어쓰지 않음. 매칭 안 된 새 거래만 시트 끝에 추가.
- **매칭 키 = 날짜(A) + 이름(B) + 금액(D)** 세 가지로만 비교. C열은 매칭 키에서 제외.
- **D1 합계** 자동 재계산.
- **헤더 배경색**: MJ 시트는 파랑, HJ 시트는 분홍. 컬럼 너비도 자동 설정.

### 보안/제약 (확실하지 않은 정보)

- **URL은 비밀**: Apps Script 웹앱 URL이 secret token 역할을 해요. 외부에 공유하지 마세요. (다만 "본인만" 액세스로 배포했다면 URL이 새도 본인 계정이 아니면 실행 불가.)
- **응답 못 읽음**: cross-origin POST라 응답을 읽지 못해요(iframe). 웹앱이 "전송 완료"라고 떠도 실제 시트 갱신 결과는 구글시트를 직접 새로고침해서 확인하세요.
- **file:// vs http://**: 파일을 더블클릭으로 열면 `file://` origin이라 일반 `fetch()`는 보안 정책상 막혀요. 그래서 이 웹앱은 hidden form + iframe POST 방식을 써요. fetch보다 안정적이지만 응답 읽기는 안 됨.
- **Apps Script 코드 업데이트 필요**: form submit 방식 지원을 위해 `apps_script.gs`가 업데이트됐어요(`json=` prefix 처리). 이미 배포한 경우 Apps Script 편집기에서 코드를 다시 붙여넣고 "**배포 → 배포 관리 → 새 버전**"으로 업데이트해주세요.
- **localStorage 저장**: 입력한 URL은 브라우저의 localStorage에 저장됨. 브라우저 캐시 비우면 다시 입력 필요. file:// 환경(파일 더블클릭) 일부 브라우저는 localStorage 거부할 수 있음.

## 제약/한계 (확실하지 않은 정보)

1. **양/음 매칭 정확성**: 같은 키의 양수·음수가 항상 환불 쌍이라고 보장은 못 함. 우연히 같은 금액의 별개 거래일 수 있음. 의심되면 옵션을 끄고 비교.
2. **이체 처리**: 카드대금 결제는 '이체'로 들어와서 기본 제외. 카드 결제 자체는 이미 '지출'로 따로 기록되어 이중계산 방지가 됨. 다만 가상의 자금흐름까지 보고 싶으면 옵션 ON.
3. **시간/타임존**: Excel 시간은 단순 텍스트로 표시. 정렬에만 사용.
4. **인터넷 연결**: SheetJS CDN(https://cdnjs.cloudflare.com) 의존. 오프라인 사용하려면 SheetJS를 같은 폴더에 받아두고 `<script src>` 수정.
